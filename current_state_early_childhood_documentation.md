# Current State: What Educators Actually Use Today

## The Three Competing Approaches

### 1. AI-Powered Generation (Mana/LoveHeart)

**What it is:** AI that generates complete learning documentation from brief inputs

**How it works:**
- Educator speaks or types short observation (2-3 sentences)
- AI generates full learning story with:
  - EYLF outcome links
  - Theorist connections (Piaget, Vygotsky, etc.)
  - Reflection
  - Next steps/planning
  - Properly formatted for assessment

**Time savings:** 4.2 hours per educator per week

**Features:**
- Voice recording while with children
- "Coach Sue" - AI assistant for pedagogical questions
- Curriculum planning generation
- Multi-language support (21 languages for families)
- Anti-fabrication guardrails (Sentinel model)
- Integrates with Storypark, OWNA, Xplor via copy-paste

**Pricing:**
- Free: 20 AI generations per month
- Personal: Usage-based ($149 for 200 requests + $0.40/request after)
- Team plans: Quote-only

**Current adoption:**
- Australian Childcare Alliance preferred partner
- 83% of users report improved educational programs
- 65% more likely to stay in sector
- Engagement scores: 6.1 → 8.5

**User feedback (from app stores):**
- "Still needs work to become user-friendly"
- "Slow to upload photos, load pages"
- "Errors frequently"
- "Has potential but currently frustrating"

**What educators say:**
- Saves massive time
- Some fear assessors will reject AI-generated content
- Concerns about losing educator voice/authenticity
- Works best when educator reviews and personalizes output

### 2. Digital Platforms (Storypark, Kinderloop, Xplor, OWNA)

**What they are:** Complete documentation + family engagement platforms

**How they work:**
- Educator logs in via web or mobile app
- Creates "stories" (learning observations) with photos/videos
- Links to EYLF outcomes manually
- Shares with families in real-time
- Families can comment, add their own "moments"
- Planning tools with templates
- Reporting/analytics for directors

**Example: Storypark workflow**
1. Take photo/video of child engaged in activity
2. Write observation (what happened, what learning occurred)
3. Select EYLF learning outcomes from dropdown
4. Add learning tags for tracking over time
5. Publish to child's portfolio (visible to families and educators)
6. Families receive notification, can comment
7. Use planning area to create next week's program based on observations
8. Link plans to stories to show cycle of planning

**Time required:**
- Still 20-40 minutes per story
- Educator must write the observation, analysis, reflection
- Must manually link frameworks
- Photos/videos must be uploaded, tagged, organized

**Features:**
- Family engagement (daily updates, photo sharing)
- Story collaboration (multiple educators can draft together)
- Planning templates
- Reporting on child progress over time
- Offline capability (write while with children, syncs later)
- Multi-room/multi-service management

**Storypark scale:**
- 11,850+ services globally
- 100,000+ educators
- 1.1 million family members

**Pricing:** Quote-only (opacity problem)

**Why educators use it:**
- Families love seeing daily updates
- Platform handles photo storage
- Directors can track who's documenting what
- Evidence portfolio builds automatically over time
- Many educators already know how to use it

**Pain points:**
- Still requires significant writing time
- Must be done during breaks or after hours
- Mobile app limited features (management tasks web-only)
- Learning curve for new educators

### 3. Paper Diaries (Butler Diaries)

**What it is:** Physical EYLF/NQS-aligned programming and reflection diaries

**How it works:**
- Weekly spread with boxes for different types of experiences
- Educator fills in experience names in appropriate boxes
- Checks EYLF outcome checkboxes to link
- Reflection spread with prompts
- Uses symbols, dates, initials to show cycle of planning
- Photo evidence section (print photos on sticker paper, stick in)

**The Butler Method:**
- Pre-fill some boxes based on previous week's reflection
- Add spontaneous experiences as they happen during the week
- Reflect at end of week using prompts
- Plan next week based on reflections
- NOT designed to fill every box every week

**Time required:**
- Still 3-5 hours/week of writing
- Must physically write observations
- Must manually link EYLF outcomes
- Must print photos separately and stick in

**Features:**
- Physical artifact trusted by assessors
- Can be shown during assessment visits
- All planning, observation, reflection in one place
- Reference pages for EYLF v2.0, MTOP v2.0, NQS
- Training included (digital guide, webinars)
- Stickers for EYLF outcomes and compliance linking

**Scale:**
- 35,000+ orders shipped
- Educators achieving "Exceeding" ratings using Butler Method

**Pricing:**
- $40-80 per diary
- $13.99 flat-rate shipping
- Completely transparent

**Why educators use it:**
- Trust physical evidence for assessment visits
- Don't have to log into anything
- Can write during nap time without computer
- Proven method with training support
- Straightforward, no tech barriers

**Pain points:**
- Still manual writing (no time savings)
- Can't share with families digitally
- Photos must be printed separately
- Doesn't update when regulations change
- Multiple books if multiple rooms

## What the Actual Workflow Looks Like

### Scenario: Educator observes child building block tower

**Manual/Butler Diary approach:**
1. Mental note or jot on paper during observation (2 min)
2. Later during break/after hours:
   - Write up observation in diary/platform (10-15 min)
   - Look up EYLF outcomes to link (5 min)
   - Decide which theorist applies (Vygotsky? Piaget?) (5 min)
   - Write reflection on what it means for child's learning (10 min)
   - Write next steps/planning (5 min)
   - Link to previous observations if building on prior learning (5 min)
3. If using Storypark: upload photos, tag child, format (5 min)
4. If using Butler: print photos on sticker paper, stick in diary (5 min)

**Total: 40-50 minutes per observation**

Do this 8-10 times per week = 5-7 hours

**Mana/AI approach:**
1. Voice memo while watching: "Emma built a tall block tower today. She counted as she stacked them and was very focused. When it fell, she said 'that's okay, I'll try again' and rebuilt it." (2 min)
2. AI generates complete learning story with EYLF links, theorist connection (Vygotsky - zone of proximal development), reflection, next steps (30 seconds)
3. Educator reviews and adjusts tone/personalization (2 min)
4. Copy-paste into Storypark or other platform (1 min)

**Total: 5 minutes per observation**

Do this 8-10 times per week = 50 minutes

**Time saved: 4-6 hours per week**

## The Pain Points in Current State

### Problem 1: Time
**Every approach except AI still requires hours of writing**
- Manual writing: 40+ min per observation
- Platform writing: 30-40 min per story
- Butler diary: 30-40 min per entry
- AI: 5 min per observation

### Problem 2: Can't step away from children
- Observations happen in moment
- Writing happens later (breaks, before/after shifts, home)
- Unpaid overtime is common
- Rushed documentation = lower quality

### Problem 3: Assessment stakes
- Assessors scrutinize documentation
- Generic/copied content fails
- Must show cycle of planning (observe → analyze → plan → implement → reflect)
- "Working Towards" rating damages reputation and enrollment
- Physical evidence (Butler) trusted more than digital by some assessors
- AI-generated content feared as inauthentic (despite Mana's guardrails)

### Problem 4: Directors can't audit quality at scale
- Multi-site operators (10+ centers) can't manually check every educator's work
- No way to know if documentation is assessment-ready before visit
- Can't track which children need more observations
- Xplor/Storypark have basic reporting but no quality analysis

### Problem 5: Family engagement expectations
- Parents want daily updates with photos
- Educators now do TWO types of documentation:
  - Compliance (for assessors)
  - Family engagement (for parents)
- Platforms combine these but double the work

### Problem 6: Cost barriers for small operators
- Family daycare educators (4-7 kids at home) can't afford $5,000+/year platforms
- Current options: expensive enterprise tools or $40-80 paper diaries
- No affordable middle ground

## The Bottom Line

**Current state in 2026:**

1. **AI (Mana) solves the time problem** - 4.2 hours saved/week
2. **Platforms (Storypark) solve the family engagement problem** - but don't save time
3. **Paper (Butler) solves the trust problem** - assessors accept physical evidence - but doesn't save time
4. **Templates don't solve any of these problems**

**What educators need:**
- Less time spent writing (Mana does this)
- Confidence their documentation will pass assessment (Butler does this)
- Family engagement without double work (Platforms do this but add work)
- Affordable options for small operators (nobody does this well)

**What Clare is offering:**
- Templates
- Monthly updates
- "Understanding WHY not just WHAT"

**The disconnect:**
- Templates still require manual writing (no time saved)
- Monthly updates don't address daily documentation burden
- Education on pedagogy doesn't reduce hours spent

Unless Clare can articulate how her toolkit saves time or solves a problem the current tools don't, she's launching a feature (templates) into a market that's already moved to AI generation (Mana), full platforms (Storypark), or trusted physical methods (Butler).
